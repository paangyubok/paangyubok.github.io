Built-in variable

Variable Name Description
CMKAE_C_COMPLIER C compiler
CMKAE_CXX_COMPLIER C++ compiler
CMAKE_LINKER the program used to link library
CMAKE_C_FLAGS C flags
CMAKE_CXX_FLAGS C++ flags
CMAKE_INCLUDE_PATH header search path. Default is null.
CMAKE_LIBRARY_PATH library search path. Default is null.
CMAKE_LIBRARY_PATH install path. Default is /usr/local
PROJECT_BINARY_DIR the path that running cmake command. As same as CMAKE_BINARY_DIR, projectname_BINARY_DIR.
PROJECT_SOURCE_DIR the root path of the project. As same as CMAKE_SOURCE_DIR, projectname_SOURCE_DIR.
CMAKE_CURRENT_SOURCE_DIR the path of current CMakeLists.txt locating
CMAKE_PREFIX_PATH semicolon-separated list of directories specifying installation prefixes to searched by the find_package(),find_program(),find_library(), find_file() and find_file().
CMAKE_BUILD_TYPE control the build type.
EXECUTABLE_OUTPUT_PATH executable location. Default is null.
LIBRARY_OUTPUT_PATH library location. Default is null.

Include

Include a directory

target_include_directories( target
    PRIVATE
    path/to/include
)

equivalent to -Ipath/to/include in gcc

options:

  • PRIVATE: the directory is added to this target’s include directories.

  • INTERFACE: the directory is added to the include directories for any targets that link this library.

  • PUBLIC: As INTERFACE, it is included in this library and also any targets that link this library.

Include third party library

find_package(lib_name version [REQUIRED] [COMPONENTS] name1 name2 ...)

find the library has installed in system.

options:

  • REQUIRED: tells the module that this is required and to fail if it can’t be found.

  • COMPONENTS: the list of components to find in the library. Can use the component as lib_name::name in the script.

Most included packages will set a variable lib_name_FOUND.

Can directly use this library without using command to include the library, but still need to link this library to the target.

The difference with find_library:

Imagine you want to use zlib in your project, you need to find the header file zlib.h, and the library libz.so (on Linux). You can use the low-level cmake commands find_path and find_library to find them, or you can use find_package(ZLIB). The later command will try to find out all what is necessary to use zlib. It can be extra macro definitions, or dependencies.

Link

Link a library

target_link_libraries( binary
    PRIVATE
    library
)

options: same as include a directory.

Compile

Compile a executable

add_executable(targets
    src/main.cc
)

Related function

  • aux_source_directory(dir, variable): find all source files in dir and store them in variable.

Compile a library

add_library( library 
    STATIC
    src/hello.cc
)

options:

  • STATIC

  • SHARED

Install

# normal install
install( TARGETS target_name
    DESTINATION folder_name)

# install library
install( TARGETS target_name
    STATIC DESTINATION folder_name)

install(move) some files to destination.

options:

    File type

  • TARGETS: install executable files(.out, .so, .a). STATIC/SHARED need to be added before DESTINATION while targets are libraries.

  • DIRECTORY: copy directory to DESTINATION and rename to folder_name.

  • FILES: copy files to DESTINATION/folder_name.

    Default destination

  • DESTINATION: default location in system. It can be edited via specify CMAKE_INSTALL_PREFIX

Message

message([option] "message text" ...)

log a message.

options:

  • STATUS: be concise which means no more than a single line, but still informative.

  • FATAL_ERROR: stop processing and generation.

  • WARNING: warning but continue processing.

string

string(REPLACE <match_string>
       <replace_string> <output_variable>
       <input> [<input>...])

Replace all occurrences of <match_string> in the <input> with <replace_string> and store the result in the <output_variable>.
