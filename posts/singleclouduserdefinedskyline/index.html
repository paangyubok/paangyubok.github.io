Towards Efficient and Privacy-Preserving User-Defined Skyline Query over Single Cloud

Songnian Zhang, Rongxing Lu

TDSC'2022

Advantage

  • Users can select attributes, choose preferences, and define constrained regions
  • Single cloud

Limit

  • Can’t dynamically update dataset
  • The complexity of this scheme directly depends on the domain of every dimension’s value. The experiment in this paper just uses $[1\sim100]$ as the domain.

Scenario

People want to find a hotel near the beach.

Multi-dimensional data: price, distance and score

Result: data which not be dominated(greater or smaller, user-defined) in every dimension

Idea

  1. Check whether a point falls within a constrained region by Multi-dimensional Point-inside-Rectangle Predicate Encryption(MPRPE)
  2. Determine skyline points by Multi-dimensional Point Dominating Predicate Encryption(MPDPE)

Improve the efficiency in step-1 by using R-tree and Hyper-Rectangle Intersection Predicate Encryption(HRIPE)

Security Model

  • Data owner: fully trusted

  • Cloud server: honest-but-curious

  • Query user: honest

Preliminaries

Block Nested Loop algorithm

Borzsony, Stephan, Donald Kossmann, and Konrad Stocker. “The skyline operator.”Â Proceedings 17th international conference on data engineering. IEEE, 2001.

Input data $X$, candidate window $W$, $\forall x\in X$:

  1. $\exist r \in W, x \prec r$. Discard x;

  2. $\exist r \in W, r\prec x$. Remove $r$ from $W$;

  3. $\forall r \in W$, $x$ is incomparable with $r$, insert $x$ into $W$;

R-tree

For 2-dimensions R-tree, the edge of rectangle consists of the minimum and the maximum of every dimension. Range query procedure is to check whether the query rectangle intersects with the non-leaf node’s rectangle.

Scheme

MPRPE

Protocol’s idea is to convert a value to a bit in a bitmap, and then multiplying to a random matrix to confuse this bitmap. Range query request also converts to continuous bits in a bitmap, and then multiplying to the inverse matrix. Comparison is checking whether their inner product is greater than 0. Comparison’s idea is using $r_er_t\cdot\Delta-r_xr_q$.

  1. Setup.

    • Input: $(\Delta,\omega)$. $\Delta={T_1,T_2,\cdots,T_d}$ is a vector that consists of the maximum value of every dimension. $\omega\ge 2$ is the number of dummy dimensions.

    • Output: $M$. $\theta=\sum_{i=1}^d{T_i}+1+\omega$. $M\in \R^{\theta\times\theta}$ is a $\theta\times \theta$-dimensional random invertible matrix.

2- Encryption.

  • Input: $(x, M, \mathbb{L}_\Delta)$.

  • Output: $||x||=\tilde{x}M$.

    Mapping a value to the bitmap and implement $r_e\cdot\Delta-r_x$.

    $$ \tilde{x}=(r_e\alpha(x^1),r_e\alpha(x^2),\cdots,-r_e,r_x^1,\cdots,r_x^\omega) $$

    $$ \alpha(x^i)_{j\in[1,T_i]}=\begin{cases}1 &\text{if } j=x^i \\ 0 &\text{Otherwise} \end{cases}, r_e>\sum{r_x^i} $$

3- Token Generation.

  • Input: $(\bold{R},M,\mathbb{L}_\Delta)$.

  • Output: $\lang q\rang=M^{-1}\widetilde{q}^T$.

    Mapping a range query to the bitmap and implement $r_t\cdot\Delta-r_q$.

    $$ \tilde{q}=(r_t\beta(R_1),r_t\beta(R_2),\cdots,d\cdot r_t,r_q^1,\cdots,r_q^\omega) $$

    $$ \beta(R_i)=(\underbrace{0,0,\cdots,0}_{R_{il}-1},1,1,\cdots,1,\underbrace{0,0,\cdots,0}_{T_i-R_{iu}}) $$

  1. Query.

$$ ||x||\cdot\lang q\rang=\tilde{x}\tilde{q}^T=r_er_t(\sum_{i=1}^d{(\alpha(x^i)\beta(R_i)^T)}-d)+\sum_{p=1}^\omega{r_x^pr_q^p} $$

Â Â Â Â Â Â Â Â If any dimension’s value is not in range, the sum of $\alpha\beta$ will be smaller than $d$. So, $\beta$ is actually a mask and sum is a AND operation.

Â Â Â Â Â Â Â Â If result is greater than 0, it means that the value of every dimension is in its corresponding range.

HRIPE

In order to solve the problem of how to judge whether two hyper-rectangle intersect in R-tree. It is similar to MPRPE.

The differences are as follow:

  • $\alpha$ combines the under bound and the upper bound of value. Use $\gamma$ to represent it as follow:

$$ \gamma(R’_i)_{j\in [1,2T_i]}=\begin{cases}1 &\text{if } j=R’_{il}\text{ or } j=R’_{iu}+T_i \\ 0 &\text{Otherwise} \end{cases} $$

  • $\beta$ combines the upper range and the under range in a bitmap. Use $\delta$ to represent it as follow:

$$ \delta(R_i)=(\underbrace{1,1,\cdots,1}_{R_{iu}},0,0,\cdots,0,\underbrace{1,1,\cdots,1}_{T_i-R_{il}+1}) $$

MPDPE

MPDPE is proposed to check dominance over encrypted data points.

  • Setup. Construct two random invertible matrices: $M, \widetilde{M}$

  • Encryption. Output two ciphertext: $||x||_L=\hat{x}M$ and $||x||_R=\tilde{x}\widetilde{M}$. Subscript $L,R$ means the value to the left(right) of $\prec$.

  • Token Generation. Output a $\theta\times\theta$ query matrix $Q$. $Q$ is related to the user preference $q_s$:

    $$ q_s^{i\in[1,d]}=\begin{cases}1&\text{if }i\text{-th-dimension is chosen and prefer }max \\ -1&\text{if }i\text{-th-dimension is chosen and prefer }min \\ 0&\text{if }i\text{-th-dimension is not chosen.}\end{cases} $$

    $$ k=\sum_{i=1}^d{|q_s^i|} $$

  • Query. Check whether $x_t\prec x_s$ using $||x||_L\circ(||x_s||_R\cdot\lang q_s\rang)$

The definitions of $\tilde{x},\hat{x},Q$ are as follow:

  • $$ \zeta(x^i)=(x^{i^2},-2x^i,1) $$

  • $$ \eta(x^i)=(\underbrace{0,0,\cdots,0}_{x^i-1},\underbrace{1,1,\cdots,1}_{T_i-x^i+1}) $$

    $$ \xi(x^i)=(1, x^i, x^{i^2}) $$

  • $$ Q_s^i=\begin{cases}I &\text{if }q_s^i=1 \\ V^i &\text{if }q_s^i=-1 \\ O &\text{if }q_s^i=0\end{cases}, V^i\in \R^{T_i\times T_i}=\begin{bmatrix}0&-1&0&\cdots&0&0 \\ 0&0&-1&\cdots&0&0 \\ \vdots&\vdots&\vdots&\ddots&\vdots&\vdots \\ 0&0&0&\cdots&0&-1 \\ 1&1&1&\cdots&1&1\end{bmatrix} $$

    $$ Q_{s’}^i=\begin{cases}I&\text{if }q_s^i=1\text{ or }-1 \\ O&\text{if }q_s^i=0\end{cases} $$

There are two parts(both sides of addition) in equation. The first part is to check whether all points satisfy $\le(\ge)$ depending on user’s preference in each dimension. The second part is to check whether existing a point satisfies $<(>)$ depending on user’s preference.

$$ r_sr_er_p>r_sr_{e’}r_{p’}>r_s\sum_{i=1}^\omega{r_x^ir_q^{i+1}} $$

$$ \alpha(x_t^\mu)(\eta(x_s^\mu)Q_s^\mu)^T=\begin{cases}1&\mu\text{-dimension is chosen and }x_t^\mu\le x_s^\mu(x_t^\mu\ge x_s^\mu) \\ 0 &\text{Otherwise}\end{cases} $$

$$ \sum_{\mu=1}^d{\zeta(x_t^\mu)(\xi(x_s^\mu)Q_{s’}^\mu)^T}=\begin{cases}>0&\exist x_t^\mu<x_s^\mu(x_t^\mu>x_s^\mu) \\ \le0&\text{Otherwise} \end{cases} $$

So, when the equation is greater than zero it means $x_t \prec x_s$

System Mechanism

  1. Data owner builds R-tree over plaintext, and then encrypts the R-tree as follow:

    • For internal nodes, execute HRIPE Encryption

    • For leaf nodes, execute MPDPE Encryption

    At last, send the encrypted R-tree to the cloud.

  2. Query user has request $(R,q_s)$. According to them, execute MPRPE Token Generation, HRIPE Token Generation and MPDPE Token Generation

  3. Cloud server uses BNL algorithm and three query algorithm to perform skyline search over encrypted R-tree.

  4. Query user has invertible matrix so it is simple to recover $x$ from $||x||_L$

Performance

The synthesized dataset has 20 dimensions with 10000 tuples, in which ten of the dimensions have a domain size of 100, and the others have a domain size of 50. Set the number of dummy dimensions $\omega=2$ .

Self Evaluation

Baseline is one by one scheme and improved scheme is R-tree based scheme.

Compare with PUSC

Search time is much faster than PUSC, although it is slower in token generation time. And the first is evaluated in second while the second is in millisecond.
